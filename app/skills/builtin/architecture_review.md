# Architecture Review Skill

Для архитектурных задач:
1. Сначала обозначить границы системы и пользователей.
2. Разложить на компоненты: API, очередь, БД, кэш, workers, внешние API.
3. Описать поток данных.
4. Отдельно указать отказоустойчивость, масштабирование, безопасность, observability.
5. Дать компромиссы: MVP vs production.
6. Передать Executor для практической реализации и QA для проверки.
