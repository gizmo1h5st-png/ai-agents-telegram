Architecture Review Skill

Для архитектурных задач:

    Сначала обозначить границы системы и пользователей.
    Разложить на компоненты: API, очередь, БД, кэш, workers, внешние API.
    Описать поток данных.
    Отдельно указать отказоустойчивость, масштабирование, безопасность, observability.
    Дать компромиссы: MVP vs production.
    Передать Executor для практической реализации и QA для проверки.
